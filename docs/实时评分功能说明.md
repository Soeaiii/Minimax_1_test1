# 大屏幕实时评分功能说明

## 功能概述

大屏幕显示页面现在支持实时评分传输，无需刷新整个页面即可获取最新的评分数据。平均分只在所有评委打分完成后显示，确保评分的公正性。

## 主要特性

### 1. 实时数据传输
- 使用 Server-Sent Events (SSE) 技术
- 每2秒自动获取最新评分数据
- 只传输评分相关数据，不影响背景图片等其他内容
- 默认关闭自动刷新，完全依赖实时数据

### 2. 智能评分显示
- **平均分控制**：只有所有评委都完成打分后才显示平均分
- **等待状态**：显示已打分评委数量和总评委数量
- **实时更新**：评委评分实时显示，无需等待

### 3. 简洁界面
- 移除了实时连接状态指示器，界面更简洁
- 保留背景加载指示器
- 专注于评分内容的展示

## 技术实现

### 后端 API
- **SSE 端点**: `/api/display/[competitionId]/stream`
- **数据格式**: JSON 格式的评分数据
- **更新频率**: 每2秒
- **连接超时**: 30秒自动关闭

### 前端组件
- **自定义 Hook**: `useScoreStream`
- **实时状态管理**: React State + useEffect
- **错误处理**: 自动重连 + 手动重连选项

## 使用方法

### 1. 大屏幕显示页面
访问 `/display/[competitionId]` 页面，系统会自动：
- 建立实时连接
- 实时更新评委评分
- 等待所有评委打分完成后显示平均分

### 2. 评分显示逻辑
- **评委评分**：实时显示每位评委的评分
- **平均分等待**：显示"等待中..."和已打分进度
- **平均分显示**：所有评委完成后显示最终平均分

## 显示状态

### 评分状态
- **等待中**: 显示黄色"等待中..."文字和打分进度
- **已完成**: 显示最终平均分数值

### 页面元素
- **评委评分**: 实时显示每位评委的评分卡片
- **平均分区域**: 根据评分完成状态显示不同内容
- **打分进度**: 显示"已打分: X/Y 位评委"
- **背景加载**: 仅在背景图片加载时显示指示器

## 数据结构

### 实时传输的数据
```typescript
interface ScoreStreamData {
  programId: string;           // 节目ID
  programName: string;         // 节目名称
  programOrder: number;        // 节目序号
  participants: Participant[]; // 参与者列表
  judgeScores: JudgeScore[];   // 评委评分
  averageScore: number | null; // 平均分（所有评委打分完成后才有值）
  allJudgesScored: boolean;    // 是否所有评委都已打分
  totalJudges: number;         // 总评委数量
  scoredJudges: number;        // 已打分评委数量
  timestamp: string;           // 时间戳
}
```

### 评委评分数据
```typescript
interface JudgeScore {
  judge: {
    id: string;
    name: string;
    avatar?: string;
  };
  totalScore: number;          // 总分
  scores: Array<{              // 详细评分
    criteriaId: string;
    criteriaName: string;
    value: number;
    weight: number;
    maxScore: number;
  }>;
}
```

## 性能优化

### 1. 数据传输优化
- 只传输必要的评分数据
- 使用 JSON 压缩格式
- 避免重复传输背景图片等大文件

### 2. 连接管理
- 页面隐藏时自动断开连接
- 30秒连接超时防止资源占用
- 智能重连避免无限重试

### 3. 前端渲染优化
- 使用 React 状态管理避免不必要的重渲染
- 动画效果使用 CSS transition
- 条件渲染减少 DOM 操作

## 故障排除

### 常见问题

1. **连接无法建立**
   - 检查比赛ID是否正确
   - 确认比赛存在且有当前节目
   - 检查网络连接状态

2. **数据不更新**
   - 查看连接状态指示器
   - 尝试手动重连
   - 检查浏览器控制台错误信息

3. **频繁断线重连**
   - 检查网络稳定性
   - 确认服务器运行正常
   - 查看服务器日志

### 调试方法

1. **浏览器开发者工具**
   - Network 标签查看 SSE 连接
   - Console 标签查看错误信息
   - Application 标签检查连接状态

2. **测试页面**
   - 使用 `/test-realtime` 页面进行调试
   - 手动控制连接状态
   - 查看详细的数据结构
