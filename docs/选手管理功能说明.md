# 选手管理功能说明

## 功能概述

选手管理系统是比赛管理平台的核心模块之一，用于管理所有参赛选手和团队的信息。该功能支持个人选手和团队的统一管理，提供完整的增删改查操作和高级筛选功能。

## 主要功能

### 1. 选手列表管理

#### 选手展示
- **卡片式布局**: 选手信息以卡片形式展示，包含头像、姓名、类型等基本信息
- **类型区分**: 通过不同颜色的标签区分个人选手（蓝色）和团队（紫色）
- **参与节目**: 显示选手参与的节目名称
- **联系信息**: 展示选手的联系方式和团队信息

#### 分页功能
- **智能分页**: 每页显示12个选手，支持前后页导航
- **页码信息**: 显示当前页码、总页数和总选手数
- **自适应布局**: 响应式设计，适配不同屏幕尺寸

### 2. 高级筛选和搜索

#### 多维度搜索
- **全文搜索**: 支持按选手姓名、简介、团队名称、联系方式搜索
- **实时搜索**: 支持回车键快速搜索
- **模糊匹配**: 不区分大小写的模糊查询

#### 类型筛选
- **全部类型**: 显示所有选手
- **个人选手**: 只显示个人参赛者
- **团队筛选**: 按具体团队名称筛选

#### 筛选控制
- **一键重置**: 快速清除所有筛选条件
- **实时刷新**: 手动刷新数据
- **筛选状态**: 保持筛选条件直到主动重置

### 3. 选手信息管理

#### 添加选手
- **表单验证**: 姓名为必填项，其他字段可选
- **灵活输入**: 支持个人选手和团队的统一添加
- **即时反馈**: 创建成功后自动刷新列表

#### 编辑选手
- **快速编辑**: 直接在列表页面打开编辑对话框
- **数据预填**: 自动填入现有信息
- **实时更新**: 保存后立即更新显示

#### 删除选手
- **安全删除**: 删除前进行二次确认
- **关联检查**: 如有关联节目则阻止删除，保护数据完整性
- **操作记录**: 所有删除操作都会记录审计日志

### 4. 选手详情页面

#### 基本信息展示
- **头像区域**: 根据类型显示不同图标
- **详细资料**: 完整显示选手的所有信息
- **时间信息**: 显示创建和更新时间

#### 参与节目管理
- **节目列表**: 显示选手参与的所有节目
- **状态展示**: 显示节目和比赛的当前状态
- **快速跳转**: 支持快速跳转到节目或比赛详情页

### 5. 统计信息

#### 实时统计
- **总选手数**: 显示系统中的总选手数量
- **个人选手**: 统计个人参赛者数量
- **团队数量**: 统计不同团队的数量
- **当前页码**: 显示当前浏览的页面

## 技术实现

### 前端架构

#### 主要组件
- **ParticipantsPage**: 主选手管理页面
- **ParticipantDetailPage**: 选手详情页面
- **表单对话框**: 添加/编辑选手的模态对话框
- **确认对话框**: 删除确认对话框

#### 状态管理
- **React Hooks**: 使用useState和useEffect管理组件状态
- **实时数据**: 支持数据的实时获取和更新
- **加载状态**: 完善的加载和错误状态处理

### 后端API

#### 主要接口
```
GET /api/participants
- 功能: 获取选手列表（支持分页和筛选）
- 参数: page, limit, search, team
- 返回: 分页的选手数据

POST /api/participants
- 功能: 创建新选手
- 权限: 需要登录
- 审计: 记录创建操作

GET /api/participants/[id]
- 功能: 获取单个选手详情
- 返回: 选手详细信息及关联节目

PUT /api/participants/[id]
- 功能: 更新选手信息
- 权限: 需要登录
- 审计: 记录更新操作

DELETE /api/participants/[id]
- 功能: 删除选手
- 权限: 需要登录
- 安全检查: 检查是否有关联节目
- 审计: 记录删除操作
```

#### 数据验证
- **必填字段**: 选手姓名不能为空
- **数据清理**: 自动去除首尾空格
- **关联检查**: 删除前检查节目关联关系

### 数据模型

#### Participant模型
```typescript
interface Participant {
  id: string;              // 选手ID
  name: string;            // 姓名/团队名称
  bio?: string;            // 简介
  team?: string;           // 团队名称（如果是团队）
  contact?: string;        // 联系方式
  createdAt: string;       // 创建时间
  updatedAt: string;       // 更新时间
  programs: Program[];     // 参与的节目
}
```

### 安全特性

#### 权限控制
- **登录验证**: 所有写操作需要用户登录
- **会话管理**: 基于NextAuth的会话验证
- **操作限制**: 根据用户角色限制操作权限

#### 审计日志
- **操作记录**: 记录所有增删改操作
- **详细信息**: 记录操作前后的数据变化
- **用户追踪**: 记录操作用户信息

#### 数据保护
- **关联保护**: 有关联节目的选手不允许删除
- **数据验证**: 严格的输入验证和数据清理
- **错误处理**: 完善的错误处理和用户提示
