# 审计日志管理功能说明

## 功能概述

审计日志管理系统是比赛管理平台的重要组成部分，用于记录和跟踪系统中所有重要的用户操作和数据变更。该功能为管理员提供了完整的系统活动追踪能力，确保系统的安全性和可审计性。

## 主要功能

### 1. 日志查看

#### 日志列表展示
- **时间轴展示**: 按时间倒序显示所有操作记录
- **操作分类**: 通过颜色编码区分不同类型的操作
  - 创建（蓝色）
  - 更新（绿色）
  - 删除（红色）
  - 评分（紫色）
  - 登录（黄色）
  - 登出（灰色）
  - 归档（橙色）
  - 恢复（青色）

#### 详细信息
- **操作用户**: 显示执行操作的用户姓名和角色
- **操作时间**: 精确到秒的时间戳
- **目标对象**: 操作涉及的具体对象ID
- **操作详情**: JSON格式的详细操作信息

### 2. 高级筛选

#### 多维度筛选
- **搜索功能**: 支持按操作类型、目标ID进行全文搜索
- **操作类型筛选**: 可按操作类型（创建、更新、删除等）筛选
- **用户筛选**: 可按特定用户筛选操作记录
- **时间范围筛选**: 支持自定义时间范围查询

#### 实时筛选
- **即时应用**: 筛选条件可即时应用，无需页面刷新
- **筛选重置**: 一键清除所有筛选条件
- **筛选状态保持**: 筛选条件在页面操作期间保持有效

### 3. 数据展示

#### 统计信息
- **总操作数**: 显示系统中的总操作记录数
- **活跃用户数**: 显示有操作记录的用户数量
- **总页数**: 显示分页后的总页数
- **当前页数**: 显示当前查看的页码

#### 分页导航
- **智能分页**: 每页显示20条记录，支持前后页导航
- **页码显示**: 显示当前页码和总页数信息
- **记录范围**: 显示当前页显示的记录范围

### 4. 详情查看

#### 日志详情对话框
- **基本信息**: 操作时间、用户信息、目标对象
- **用户信息**: 用户姓名、邮箱、角色标识
- **操作详情**: 格式化显示的详细操作信息
- **元数据**: 日志ID、用户ID等技术信息

#### 便捷操作
- **复制功能**: 支持复制目标ID和详细信息
- **格式化显示**: JSON数据格式化显示，便于阅读
- **响应式设计**: 适配不同屏幕尺寸

### 5. 数据导出

#### CSV导出
- **全量导出**: 支持导出所有匹配筛选条件的记录
- **筛选导出**: 尊重当前筛选条件，只导出符合条件的记录
- **文件命名**: 自动生成带日期的文件名
- **数据完整性**: 导出包含所有字段信息

#### 导出字段
- 操作时间
- 用户姓名
- 用户邮箱
- 操作类型
- 目标对象ID
- 详细信息

## 技术实现

### 后端API

#### 主要接口
```
GET /api/audit-logs
- 功能: 获取审计日志列表
- 参数: page, limit, search, action, userId, startDate, endDate, export
- 返回: 分页的日志数据或CSV文件
```

#### 权限控制
- **管理员专用**: 只有ADMIN角色用户可以访问审计日志
- **权限验证**: 基于NextAuth会话验证用户权限
- **安全防护**: 防止未授权访问敏感日志信息

### 前端组件

#### 主要组件
- **AuditLogsPage**: 主页面组件，包含所有功能
- **AuditLogDetailDialog**: 详情对话框组件
- **筛选面板**: 集成的筛选控制面板

#### 状态管理
- **React Hooks**: 使用useState和useEffect管理组件状态
- **实时更新**: 支持数据的实时刷新和更新
- **用户体验**: 加载状态、错误处理、空数据提示

## 审计日志类型

### 用户操作
- **LOGIN**: 用户登录操作
- **LOGOUT**: 用户登出操作

### 数据操作
- **CREATE**: 创建新记录（比赛、节目、参与者等）
- **UPDATE**: 更新现有记录
- **DELETE**: 删除记录

### 特殊操作
- **SCORE**: 评分操作
- **ARCHIVE**: 归档操作
- **RESTORE**: 恢复操作

## 日志详情格式

### 登录日志
```json
{
  "ip": "192.168.1.100",
  "userAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36"
}
```

### 创建操作
```json
{
  "name": "比赛名称",
  "description": "创建了新的比赛",
  "type": "Competition"
}
```

### 更新操作
```json
{
  "changes": {
    "status": { "from": "PENDING", "to": "ACTIVE" }
  },
  "type": "Competition"
}
```

### 评分操作
```json
{
  "programName": "节目名称",
  "scores": {
    "技术难度": 8.5,
    "艺术表现": 9.0,
    "整体印象": 8.8
  },
  "type": "Score"
}
```

## 安全考虑

### 数据保护
- **敏感信息过滤**: 不记录密码等敏感信息
- **访问控制**: 严格的权限验证机制
- **数据完整性**: 确保日志数据的不可篡改性

### 隐私保护
- **最小化原则**: 只记录必要的审计信息
- **数据匿名化**: 在适当情况下对敏感数据进行匿名化处理
- **合规性**: 符合数据保护相关法规要求

## 性能优化

### 数据库优化
- **索引策略**: 在用户ID、操作类型、时间戳字段上建立索引
- **分页查询**: 使用高效的分页机制减少数据传输
- **查询优化**: 优化复杂查询，提高响应速度

### 前端优化
- **懒加载**: 按需加载日志数据
- **缓存策略**: 合理缓存查询结果
- **虚拟滚动**: 对大量数据使用虚拟滚动技术

### 日志轮转

## 使用指南

### 管理员操作流程
1. **访问**: 以管理员身份登录后访问审计日志页面
2. **查看**: 浏览系统中的所有操作记录
3. **筛选**: 根据需要设置筛选条件查看特定记录
4. **详情**: 点击查看按钮查看操作的详细信息
5. **导出**: 需要时导出日志数据用于分析或归档

### 故障排查
1. **操作追踪**: 通过日志追踪特定操作的执行情况
2. **用户行为**: 分析用户的操作模式和行为
3. **系统分析**: 通过日志分析系统的使用情况和性能

## 注意事项

### 数据库配置
- **副本集要求**: MongoDB需要配置为副本集以支持事务
- **权限设置**: 确保数据库用户具有相应的读写权限
- **备份策略**: 定期备份审计日志数据

### 系统要求
- **存储空间**: 确保有足够的存储空间用于日志数据
- **网络带宽**: 大量日志查询可能占用较多带宽
- **处理能力**: 确保服务器有足够的处理能力处理日志查询

通过这个全面的审计日志管理系统，管理员可以有效地监控和追踪系统中的所有重要操作，确保系统的安全性和合规性。
