# 智能切换自动跳转功能优化

## 功能概述

优化智能切换功能，让它不仅能切换节目状态，还能在节目完成时自动切换到下一个节目，提升比赛管理的流畅性。

## 功能改进

### 优化前
- **智能切换**：只切换当前节目的状态（等待中→进行中→已完成→等待中）
- **节目切换**：需要手动在节目列表中选择下一个节目
- **操作流程**：状态切换 + 手动节目切换（两步操作）

### 优化后
- **智能切换**：状态切换 + 自动节目跳转
- **自动跳转逻辑**：当节目从"进行中"变为"已完成"时，自动切换到下一个节目
- **操作流程**：一键完成状态切换和节目跳转

## 技术实现

### 核心逻辑
```tsx
const smartStatusUpdate = (programId: string) => {
  // ... 原有状态切换逻辑 ...
  
  // 智能节目切换：当节目从"进行中"变为"已完成"时，自动切换到下一个节目
  if (currentStatus === 'PERFORMING' && newStatus === 'COMPLETED') {
    // 按order排序找到下一个节目
    const sortedPrograms = [...displayData.programs].sort((a, b) => a.order - b.order);
    const currentIndex = sortedPrograms.findIndex(p => p.id === programId);
    
    if (currentIndex >= 0 && currentIndex < sortedPrograms.length - 1) {
      // 有下一个节目，切换到下一个节目
      const nextProgram = sortedPrograms[currentIndex + 1];
      form.setValue('currentProgramId', nextProgram.id);
      
      // 自动将下一个节目设置为"进行中"
      setPendingProgramStatus(nextProgram.id, 'PERFORMING');
    }
    // 如果是最后一个节目，保持在当前节目
  }
};
```

### 关键特性

1. **触发条件**：只在节目从"进行中"变为"已完成"时触发自动跳转
2. **排序逻辑**：按节目的 `order` 字段排序，确保按正确顺序跳转
3. **边界处理**：如果是最后一个节目，保持在当前节目，不会跳转
4. **状态同步**：自动将下一个节目设置为"进行中"状态

## 应用场景

### 适用场景
1. **比赛现场管理**：主持人或技术人员可以专注于比赛进程，减少操作干扰
2. **大型比赛**：节目数量多时，自动跳转显著提升效率
3. **实时直播**：减少操作时间，保持直播流畅性

### 特殊情况处理
1. **最后一个节目**：完成后不会跳转，保持在当前节目
2. **非连续操作**：只在"进行中→已完成"时触发，其他状态切换不影响
3. **手动干预**：用户仍可手动选择任意节目，不受自动跳转影响

## 技术细节

### 状态管理
- 使用 `form.setValue('currentProgramId', nextProgram.id)` 更新当前选中节目
- 使用 `setPendingProgramStatus(nextProgram.id, 'PERFORMING')` 设置下一个节目状态
- 保持与现有状态管理系统的兼容性

### 排序算法
- 使用 `sort((a, b) => a.order - b.order)` 确保按节目顺序排列
- 使用 `findIndex` 找到当前节目在排序后数组中的位置
- 边界检查确保不会越界访问

### 性能考虑
- 排序操作只在触发时执行，不影响常规操作性能
- 使用展开运算符 `[...displayData.programs]` 避免修改原数组
