# 裁判打分页面优化

## 优化内容

### 1. 浏览器切换状态保持
- **问题**: 裁判切换浏览器标签页或应用后，再回到打分页面会回到第一个节目
- **解决方案**: 使用localStorage保存当前节目索引，实现状态持久化

#### 实现机制
1. **本地存储**: 使用`localStorage`保存当前节目索引
2. **多重触发保存**:
   - 页面可见性变化时（切换标签页）
   - 页面卸载前（关闭浏览器）
   - 手动切换节目时
   - URL参数变化时

3. **智能恢复**:
   - 优先级：URL参数 > 本地存储 > 默认值(0)
   - 自动验证索引有效性
   - 异常处理，避免崩溃
   - URL参数同步：切换节目时自动更新URL参数

#### 核心代码
```typescript
// 获取存储键名
const getStorageKey = () => `judge_scoring_${competitionId}_program_index`;

// 保存当前节目索引
const saveCurrentProgramIndex = (index: number) => {
  try {
    localStorage.setItem(getStorageKey(), index.toString());
  } catch (error) {
    console.warn('Failed to save program index to localStorage:', error);
  }
};

// 监听页面可见性变化
useEffect(() => {
  const handleVisibilityChange = () => {
    if (document.visibilityState === 'hidden') {
      saveCurrentProgramIndex(currentProgramIndex);
    }
  };
  document.addEventListener('visibilitychange', handleVisibilityChange);
  return () => document.removeEventListener('visibilitychange', handleVisibilityChange);
}, [currentProgramIndex]);

// URL参数同步
const updateUrlParam = (index: number) => {
  const newUrl = new URL(window.location.href);
  newUrl.searchParams.set('program', index.toString());
  window.history.replaceState({}, '', newUrl.toString());
};
```

### 2. 手机端显示优化

#### 响应式布局改进
1. **容器间距**: `p-6` → `p-4 sm:p-6`
2. **标题大小**: `text-3xl` → `text-2xl sm:text-3xl`
3. **卡片布局**: 垂直优先，大屏幕水平排列

#### 节目信息卡片优化
- **标题区域**: 垂直堆叠 → 响应式flex布局
- **参赛者标签**: 独立行显示，支持换行
- **状态徽章**: 自适应定位

#### 评分表单优化
1. **评分标准卡片**:
   - 标题和权重信息垂直排列
   - 分数输入和评语在大屏幕并排，小屏幕堆叠
   - 输入框字体大小优化（`text-base`）

2. **操作按钮布局**:
   - 手机端：垂直排列，全宽按钮
   - 桌面端：水平排列，自适应宽度
   - 按钮顺序优化：保存按钮在手机端置顶

## 技术实现

### 存储策略
- **键名格式**: `judge_scoring_{competitionId}_program_index`
- **数据类型**: 字符串存储数字索引
- **异常处理**: try-catch包装，静默失败

### 事件监听
1. **visibilitychange**: 页面可见性变化
2. **beforeunload**: 页面卸载前
3. **组件卸载**: useEffect清理函数

### 响应式设计
- **断点**: 使用Tailwind的`sm:`前缀（640px+）
- **布局**: Flexbox + CSS Grid
- **优先级**: 移动端优先设计
