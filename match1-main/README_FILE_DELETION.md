# 文件删除功能改进

## 功能概述

现在在删除文件时，系统会同时删除数据库记录和本地存储的物理文件，确保存储空间不会浪费。

## 主要改进

### 1. 单个文件删除 (新增)

- **API**: `DELETE /api/files/[id]`
- **功能**: 删除指定ID的单个文件
- **安全检查**:
  - 用户身份验证
  - 权限检查（如果文件被使用，需要管理员或组织者权限）
  - 文件存在性验证

### 2. 批量文件删除 (增强)

- **API**: `DELETE /api/files`
- **功能**: 批量删除多个文件
- **增强内容**:
  - 添加了物理文件删除
  - 权限检查和审计日志
  - 错误处理和详细的反馈信息

## 实现细节

### 物理文件删除流程

1. **验证权限**: 检查用户是否有删除权限
2. **获取文件信息**: 从数据库获取文件详细信息
3. **检查文件关联**: 验证文件是否被节目或比赛使用
4. **删除物理文件**: 使用 `fs.unlink()` 删除文件系统中的文件
5. **删除数据库记录**: 从数据库中移除文件记录
6. **记录审计日志**: 记录删除操作的详细信息

### 权限控制

- **普通用户**: 只能删除未被使用的文件
- **管理员/组织者**: 可以删除任何文件，包括正在使用的文件

### 错误处理

- **物理文件不存在**: 记录警告但继续删除数据库记录
- **权限不足**: 返回适当的错误信息
- **文件正在使用**: 检查关联关系并给出明确提示

## API 接口

### 删除单个文件

```http
DELETE /api/files/{fileId}
Authorization: Bearer <token>
```

**响应**:
```json
{
  "message": "文件删除成功",
  "deletedFile": {
    "id": "file_id",
    "filename": "example.jpg"
  }
}
```

### 批量删除文件

```http
DELETE /api/files
Content-Type: application/json
Authorization: Bearer <token>

{
  "fileIds": ["file1_id", "file2_id", "file3_id"]
}
```

**响应**:
```json
{
  "message": "成功删除 3 个文件",
  "deletedCount": 3,
  "physicalDeleteErrors": []
}
```

## 审计日志

所有文件删除操作都会记录到审计日志中，包含以下信息：

- 操作用户ID
- 操作时间
- 被删除文件的详细信息（文件名、路径、大小、类型）
- 是否为批量操作

## 使用场景

### 1. 文件管理页面
- 支持选择多个文件批量删除
- 实时反馈删除进度和结果

### 2. 节目附件管理
- 单个文件删除功能
- 自动刷新文件列表

### 3. 背景图片管理
- 移除背景时不删除文件（只是取消使用）
- 文件仍可在文件管理中删除

## 安全特性

1. **身份验证**: 所有删除操作都需要用户登录
2. **权限控制**: 基于用户角色的删除权限
3. **关联检查**: 防止删除正在使用的重要文件
4. **审计跟踪**: 完整的操作记录
5. **容错处理**: 即使物理删除失败也会清理数据库记录

## 注意事项

1. **不可逆操作**: 文件删除后无法恢复，请谨慎操作
2. **存储同步**: 确保数据库和文件系统的一致性
3. **权限提醒**: 删除被使用的文件需要相应权限
4. **错误恢复**: 系统会尽力保持数据一致性，即使部分操作失败
