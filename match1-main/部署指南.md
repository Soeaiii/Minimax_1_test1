## 🛠 环境要求

### 系统要求
- **操作系统**：Linux (Ubuntu 20.04+)、macOS、Windows 10+
- **Node.js**：18.0+ 或 20.0+
- **npm**：9.0+ 或 pnpm 8.0+
- **数据库**：MongoDB 7.0+
- **内存**：最少2GB，推荐4GB+
- **存储**：最少10GB可用空间

### 开发工具
- **代码编辑器**：VS Code、WebStorm等
- **Git**：版本控制工具
- **浏览器**：Chrome、Firefox、Safari、Edge（最新版本）

## 🚀 快速部署

### 1. 克隆项目
```bash
git clone <项目仓库地址>
cd match7/judge-system
```

### 2. 安装依赖
```bash
# 使用 npm
npm install

# 或使用 pnpm（推荐）
pnpm install
```

### 3. 环境配置
创建 `.env` 文件：
```bash
cp .env.example .env
```

编辑 `.env` 文件：
```env
# 数据库连接
DATABASE_URL="mongodb://localhost:27017/match7"

# NextAuth 配置
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your-secret-key-here"

# 文件上传目录
UPLOAD_DIR="./public/uploads"
```

### 4. 数据库初始化
```bash
# 生成 Prisma 客户端
npx prisma generate

# 初始化种子数据
npm run prisma:seed
```

### 5. 启动应用
```bash
# 开发模式
npm run dev

# 生产模式
npm run build
npm start
```

## 🗄 数据库配置

### MongoDB 安装 (Ubuntu)

#### 1. 添加 MongoDB 仓库
```bash
# 导入公钥
curl -fsSL https://www.mongodb.org/static/pgp/server-7.0.asc | sudo gpg -o /usr/share/keyrings/mongodb-server-7.0.gpg --dearmor

# 添加仓库
echo "deb [ arch=amd64,arm64 signed-by=/usr/share/keyrings/mongodb-server-7.0.gpg ] https://repo.mongodb.org/apt/ubuntu jammy/mongodb-org/7.0 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-7.0.list
```

#### 2. 安装 MongoDB
```bash
sudo apt-get update
sudo apt-get install -y mongodb-org
```

#### 3. 启动服务
```bash
# 启动 MongoDB
sudo systemctl start mongod

# 设置开机自启
sudo systemctl enable mongod

# 检查状态
sudo systemctl status mongod
```

### MongoDB 安装 (macOS)
```bash
# 使用 Homebrew
brew tap mongodb/brew
brew install mongodb-community@7.0

# 启动服务
brew services start mongodb/brew/mongodb-community
```

### 数据库连接测试
```bash
# 连接到 MongoDB
mongosh

# 显示数据库列表
show dbs

# 退出
exit
```

## ⚙️ 环境变量配置

### .env 文件详解

```env
# ===========================================
# 数据库配置
# ===========================================
DATABASE_URL="mongodb://localhost:27017/match7"
# 格式：mongodb://[用户名:密码@]主机地址:端口/数据库名

# ===========================================
# NextAuth.js 认证配置
# ===========================================
NEXTAUTH_URL="http://localhost:3000"
# 生产环境改为实际域名，如: https://your-domain.com

NEXTAUTH_SECRET="your-super-secret-key-change-this"
# 生产环境必须使用强密码，可使用：openssl rand -base64 32

# ===========================================
# 文件上传配置
# ===========================================
UPLOAD_DIR="./public/uploads"
# 文件存储目录，确保目录存在且有写权限

# ===========================================
# 应用配置（可选）
# ===========================================
NODE_ENV="development"
# 环境模式：development | production

PORT=3000
# 服务端口（默认3000）

# ===========================================
# 邮件配置（未来扩展）
# ===========================================
# SMTP_HOST="smtp.gmail.com"
# SMTP_PORT=587
# SMTP_USER="your-email@gmail.com"
# SMTP_PASS="your-app-password"
```

### 生成安全密钥
```bash
# 生成 NextAuth Secret
openssl rand -base64 32

# 或使用 Node.js
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

## 🌐 生产环境部署

### 1. 服务器准备

#### 更新系统
```bash
sudo apt update && sudo apt upgrade -y
```

#### 安装必要软件
```bash
# 安装 Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 pm2 进程管理器
sudo npm install -g pm2

# 安装 nginx
sudo apt install nginx -y
```

### 2. 应用部署

#### 构建应用
```bash
cd /path/to/your/app
npm install
npm run build
```

#### PM2 配置
创建 `ecosystem.config.js`：
```javascript
module.exports = {
  apps: [{
    name: 'match7-judge-system',
    script: 'npm',
    args: 'start',
    instances: 'max',
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production',
      PORT: 3000
    },
    env_production: {
      NODE_ENV: 'production',
      PORT: 3000
    }
  }]
};
```

#### 启动应用
```bash
# 启动应用
pm2 start ecosystem.config.js --env production

# 保存配置
pm2 save

# 设置开机自启
pm2 startup
```

### 3. Nginx 反向代理

#### 配置文件
创建 `/etc/nginx/sites-available/match7`：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态文件缓存
    location /_next/static {
        alias /path/to/your/app/.next/static;
        expires 365d;
        access_log off;
    }

    # 上传文件
    location /uploads {
        alias /path/to/your/app/public/uploads;
        expires 30d;
        access_log off;
    }
}
```

#### 启用站点
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/match7 /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 nginx
sudo systemctl restart nginx
```

### 4. SSL 证书配置

#### 使用 Let's Encrypt
```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo crontab -e
# 添加：0 3 * * * /usr/bin/certbot renew --quiet
```

### 5. 防火墙配置
```bash
# 启用防火墙
sudo ufw enable

# 允许必要端口
sudo ufw allow 22    # SSH
sudo ufw allow 80    # HTTP
sudo ufw allow 443   # HTTPS

# 检查状态
sudo ufw status
```

## 🔧 常见问题

### 1. 数据库连接问题

**Q: 无法连接到 MongoDB**
```bash
# 检查 MongoDB 状态
sudo systemctl status mongod

# 查看错误日志
sudo tail -f /var/log/mongodb/mongod.log

# 重启服务
sudo systemctl restart mongod
```

**Q: 权限错误**
```bash
# 检查数据目录权限
sudo chown -R mongodb:mongodb /var/lib/mongodb
sudo chown mongodb:mongodb /tmp/mongodb-*.sock
```

### 2. 应用启动问题

**Q: npm install 失败**
```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

**Q: 端口被占用**
```bash
# 查看端口占用
sudo lsof -i :3000

# 杀死进程
sudo kill -9 <PID>
```

### 3. 权限问题

**Q: 文件上传失败**
```bash
# 创建上传目录
mkdir -p public/uploads

# 设置权限
chmod 755 public/uploads

# 设置所有者（如果需要）
chown www-data:www-data public/uploads
```

### 4. 性能优化

**Q: 应用响应缓慢**
- 检查服务器资源使用情况
- 优化数据库查询
- 启用 nginx 缓存
- 使用 CDN 加速静态资源

**监控命令**：
```bash
# 查看系统资源
htop

# 查看应用状态
pm2 status
pm2 logs match7-judge-system

# 查看数据库性能
mongosh --eval "db.stats()"
```

