# 保存按钮位置和状态保持优化

## 优化概述

将大屏幕管理页面中的保存按钮从底部移动到智能切换按钮旁边，并优化保存后的页面状态保持功能。

## 优化前后对比

### 优化前
- **保存按钮位置**：页面底部的全宽按钮
- **保存逻辑**：保存后重新加载整个页面数据
- **用户体验**：保存后页面会刷新，丢失当前选择状态

### 优化后
- **保存按钮位置**：智能切换按钮旁边，只在有未保存更改时显示
- **保存逻辑**：保存后更新本地状态，保持页面状态
- **用户体验**：保存后页面状态保持不变，操作更流畅

## 具体改进

### 1. 保存按钮位置优化

#### 移动到智能切换旁边
```tsx
// 在智能切换按钮后添加保存按钮
<Button
  type="button"
  variant="secondary"
  size="sm"
  onClick={() => smartStatusUpdate(currentProgram.id)}
  disabled={false}
  className="flex items-center gap-1"
  title="智能切换：等待中→进行中→已完成→等待中"
>
  <Settings className="h-3 w-3" />
  智能切换
</Button>
{hasUnsavedChanges && (
  <Button
    type="submit"
    variant="default"
    size="sm"
    disabled={saving}
    className="flex items-center gap-1"
  >
    <Save className="h-3 w-3" />
    {saving ? '保存中...' : '保存'}
  </Button>
)}
```

#### 移除底部保存按钮
- 删除了页面底部的全宽保存按钮
- 更新了提示文字，指向新的保存按钮位置

### 2. 状态保持优化

#### 优化前的保存逻辑
```tsx
// 保存后重新加载数据
setSuccessMessage('设置保存成功！');
await fetchDisplayData(); // 重新加载整个页面数据
```

#### 优化后的保存逻辑
```tsx
// 更新本地状态而不是重新加载数据
if (hasUnsavedChanges) {
  // 更新本地节目状态
  setDisplayData(prevData => {
    if (!prevData) return prevData;
    
    const updatedPrograms = prevData.programs.map(program => {
      if (pendingStatusUpdates[program.id]) {
        return {
          ...program,
          currentStatus: pendingStatusUpdates[program.id]
        };
      }
      return program;
    });
    
    return {
      ...prevData,
      programs: updatedPrograms,
      settings: {
        ...prevData.settings,
        ...data
      }
    };
  });
  
  // 清除待保存的状态更新
  setPendingStatusUpdates({});
}
```

### 3. 用户界面改进

#### 按钮显示逻辑
- **智能切换按钮**：始终显示
- **保存按钮**：只在有未保存更改时显示（`hasUnsavedChanges`）
- **按钮样式**：保存按钮使用主色调（`variant="default"`），突出重要性

#### 提示文字更新
- **状态提示**：从"点击下方保存按钮"改为"点击右侧保存按钮"
- **未保存更改提示**：从"点击下方'保存节目设置'按钮"改为"点击智能切换旁边的'保存'按钮"

## 技术实现细节

### 1. 状态管理优化
- 使用 `setDisplayData` 直接更新本地状态
- 避免不必要的网络请求和页面重新渲染
- 保持用户的当前选择和操作状态

### 2. 用户体验提升
- **即时反馈**：保存按钮只在需要时出现
- **位置就近**：保存按钮靠近操作区域
- **状态保持**：保存后不丢失当前页面状态

### 3. 性能优化
- **减少网络请求**：避免保存后的数据重新加载
- **减少渲染**：只更新必要的状态
- **提升响应速度**：本地状态更新比网络请求更快

## 用户操作流程

### 优化前
1. 用户修改节目状态
2. 滚动到页面底部点击保存按钮
3. 页面重新加载数据
4. 用户需要重新选择节目和滚动到操作位置

### 优化后
1. 用户修改节目状态
2. 在当前操作区域直接点击保存按钮
3. 状态立即更新，页面保持当前状态
4. 用户可以继续操作，无需重新定位

## 优化效果

### ✅ 用户体验提升

### ✅ 性能提升

- **逻辑分组**：相关操作按钮放在一起
