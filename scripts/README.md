# 测试数据生成脚本使用说明

## 📋 脚本功能

`generate-test-data.js` 脚本用于生成完整的比赛管理系统测试数据，包括：

- 🏆 **3个比赛**：校园歌手大赛、舞蹈艺术节、器乐演奏比赛
- 👥 **5个评委用户**：不同专业领域的评委
- 🎭 **18个参赛者**：分布在不同比赛中
- 🎪 **18个节目**：每个比赛6个节目，不同状态
- 📋 **评分标准**：针对不同比赛类型的专业评分标准
- 💯 **评分记录**：已完成节目的完整评分数据
- 🥇 **排名数据**：基于评分计算的排名结果
- 🖥️ **显示设置**：大屏幕显示的完整配置

## 🚀 使用方法

### 1. 安装依赖

确保已安装必要的依赖：

```bash
cd judge-system
npm install bcryptjs @prisma/client
```

### 2. 运行脚本

```bash
# 方法1：直接运行脚本
node scripts/generate-test-data.js

# 方法2：使用 npm scripts（如果在 package.json 中配置）
npm run generate-data
```

### 3. 检查生成结果

脚本运行后会输出详细的生成日志：


## 🛠️ 脚本特性

### 智能数据生成

- **关系完整性**：确保所有数据表之间的关系正确
- **状态一致性**：节目状态、比赛状态、评分记录保持一致
- **权重计算**：自动计算加权平均分和排名
- **随机性**：评分数据具有合理的随机性

### 安全特性

- **不覆盖用户**：保留现有用户数据，只补充必要的评委
- **清理机制**：仅清理测试相关数据，不影响系统配置
- **错误处理**：完整的异常处理和日志输出

### 灵活配置

- **可重复运行**：支持多次运行，每次会清理旧数据
- **模块化设计**：各功能模块独立，便于维护
- **数据一致性**：确保生成的数据符合业务逻辑

## 🔧 自定义配置

### 修改比赛数据

编辑 `createCompetitions` 函数中的比赛信息：

```javascript
const competitions = [
  {
    name: '你的比赛名称',
    description: '比赛描述',
    startTime: new Date('2024-04-01T09:00:00Z'),
    endTime: new Date('2024-04-01T18:00:00Z'),
    status: 'ACTIVE', // PENDING, ACTIVE, FINISHED, ARCHIVED
    rankingUpdateMode: 'REALTIME' // REALTIME, BATCH
  }
];
```

### 修改参赛者数据

编辑 `createParticipants` 函数中的参赛者信息：

```javascript
const participantsData = [
  { 
    name: '参赛者姓名', 
    bio: '个人简介', 
    team: '所属团队', 
    contact: '联系方式' 
  }
];
```

### 修改评分标准

编辑 `createScoringCriteria` 函数中的评分标准：

```javascript
const criteriaByType = {
  yourType: [
    { name: '标准名称', weight: 0.4, maxScore: 10 },
    // 权重总和应为 1.0
  ]
};
```
